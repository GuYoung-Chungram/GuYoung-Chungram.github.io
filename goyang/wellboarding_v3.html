보고서 품질을 결정하는 3개 층위
🏗️
🎯
Layer 1 · 가장 효과 큼
맥락 정보 (Context)
AI는 독자가 누군지, 왜 이 보고서를 쓰는지 전혀 모릅니다. 이 정보를 주면 어조·분량·강조점이 즉시 달라집니다.
📋 결재자 직급
과장 / 국장 / 부시장 / 시장
직급이 높을수록 → 더 짧고 결론 중심
🎯 보고 목적
현황 공유 / 결재 요청 / 정책 제안
목적이 바뀌면 결론 위치가 바뀜
⏱️ 긴급도
즉시 / 이번 주 / 정기 보고
긴급할수록 → 두괄식·압축형
🏛️ 기관 유형
광역 / 기초 / 중앙부처
예산 표기·법령 인용 방식 달라짐
📐
Layer 2 · 즉각 체감
출력 제약 정보 (Format Control)
"공문서 문체 사용"은 너무 추상적입니다. 구체적 규칙을 줄수록 AI가 정확하게 따릅니다.
📏 분량 기준
A4 1매 이내 (약 700자)
글자 수 지정하면 오버 방지
🚫 금지어 목록
매우, 상당히, 다양한, 적극적으로, 철저히
공문서다운 문체 즉시 확보
🔢 수치 표기 규칙
증가 ▲, 감소 △, 단위 반드시 병기
행안부 공문서 규정 준수
📝 문체 지정
~임, ~됨, ~할 예정임 체 통일
공문서 어투 일관성 유지
📚
Layer 3 · 고급 수강생용
레퍼런스 정보 (Reference)
AI에게 실제 데이터·법령·사례를 넘겨주면 결과물이 한 단계 점프합니다. 아래 링크에서 고양시 공식 자료를 바로 가져오세요.
📄 우수 보고서 샘플
기관 내 잘 쓴 보고서 일부를 붙여넣기
"아래 형식을 참고해서 작성해줘"
⚖️ 관련 법령·지침명
지방자치법 제○조, 행안부 지침 명시
AI가 법적 근거 자동 인용
📊 전년도 비교 수치
전년 대비 분석 데이터 함께 입력
증감 분석이 자동 생성됨
📢 실제 민원 내용
실제 민원 텍스트 그대로 붙여넣기
문제의식이 정확히 반영됨

예시 1 · 현황 보고서 (결재자: 국장, 목적: 예산 증액 설득)
## 맥락 정보 (Layer 1) [결재자 직급]: 국장 [보고 목적]: 노인복지 예산 증액을 위한 결재 요청 [기관]: 고양특례시 복지국 [긴급도]: 이번 주 결재 필요 ## 보고 내용 (Layer 2 — 형식 제약 포함) [보고 주제]: 2026년 고양특례시 고령인구 현황 [기준일]: 2026. 3. 31. [주요 수치]: {KOSIS에서 가져온 고령인구 통계} [전년 비교]: {전년도 수치} [출력 형식]: - 분량: A4 1매 이내 (600~750자) - 수치 표기: 증가 ▲, 감소 △, 단위 병기 - 문체: ~임, ~됨 체 통일 - 금지어: 매우, 상당히, 다양한, 적극적으로, 철저히 - 제목: 【 】 형식, 기준일 포함 ## 레퍼런스 (Layer 3) 아래 법령 근거를 보고서에 자연스럽게 인용해줘: 「노인복지법」 제4조 (국가 및 지방자치단체의 책임) 「고양특례시 노인복지 지원 조례」 제○조 # 위 통계는 KOSIS(kosis.kr)에서 복사해서 붙여넣기하세요.

예시 2 · 정책 제안서 (결재자: 과장, 목적: 사업 신규 추진 승인)
## 맥락 정보 (Layer 1) [결재자 직급]: 과장 [보고 목적]: 신규 사업 추진 승인 요청 [기관]: 고양특례시 도시재생과 [긴급도]: 정기 보고 (다음 주 팀장 회의 자료) ## 보고 내용 (Layer 2) [제안 주제]: {제안하려는 사업명} [현재 문제점]: {현행 문제 2~3가지 + 수치 근거} [제안 내용]: {핵심 제안 사항} [소요 예산]: {국비 ○○%, 시비 ○○% 형식으로} [추진 일정]: {분기별 일정} [출력 형식]: - 분량: A4 2매 (2,800~3,200자) - 예산: 총 ○○백만원(국비 ○○%, 시비 ○○%) 형식 - 금지: 주관적 형용사, 추상적 기대효과 표현 - 기대효과: 반드시 정량 지표로 표현 ## 레퍼런스 (Layer 3) 타 지자체 유사 사업 사례: {ALIO 또는 KOSIS에서 찾은 사례 1~2건 붙여넣기} # 사례를 넣으면 AI가 "수원시 사례와 비교하면~" 형식으로 자동 인용합니다.

보고서 자동화하기
매번 같은 형식의 보고서를 AI에게 반복 설명하는 대신, 한 번만 설정하면 항상 일관된 결과를 만드는 방법을 배웁니다. Google Gemini의 Gems 또는 Claude의 Projects를 활용합니다.
왜 자동화가 필요한가?
🎯
😤
지금의 문제
매번 보고서 형식을 설명하고, 공문서 규칙을 다시 알려주고, 결과물이 들쭉날쭉하여 수정에 시간을 씁니다.
✅
자동화 후의 변화
담당자는 핵심 데이터만 입력하면 됩니다. 형식·어조·구조는 AI가 항상 동일하게 유지합니다.
🏛️
현장에서의 가치
부서 내 보고서 품질이 균일해지고, 신규 직원도 베테랑 수준의 형식을 바로 활용할 수 있습니다.
🔄
지속 개선
지침(시스템 인스트럭션)을 한 번만 수정하면 이후 모든 결과물에 즉시 반영됩니다.
어떤 도구를 쓸까? — Gems vs Claude Projects
⚖️
구분 Google Gemini Gems Claude Projects
접근 방법 gemini.google.com → Gems 탭에서 새 Gem 생성 claude.ai → Projects에서 새 프로젝트 생성
지침 설정 Gem 설명란에 시스템 인스트럭션 작성 (500~2,000자 권장) Project Instructions에 지침 작성 (최대 2,000 토큰)
적합한 용도 Gems 특정 보고서 1종에 특화, 반복 사용이 잦을 때 Claude 다양한 보고서를 하나의 프로젝트에서 관리할 때
파일 첨부 대화 중 파일 업로드 가능 프로젝트에 참조 문서 영구 첨부 가능
공유 Gem 링크로 부서원과 공유 가능 프로젝트 공유 (유료 플랜)
추천 대상 Google Workspace 사용 기관 개인 심화 활용자
💡
어느 쪽이든 핵심은 같습니다 — "시스템 인스트럭션(지침)"을 잘 작성하는 것입니다. 도구는 달라도 지침 작성법은 동일합니다.
Gems / Projects 구성 순서
🛠️
1
보고서 유형 선택 & 목표 정의
어떤 보고서를 자동화할지 결정합니다. 한 Gem = 한 보고서 유형이 원칙입니다.
예) "민원 처리 결과 보고서" 전용 Gem, "예산 집행 현황 보고서" 전용 Gem
현황 보고서정책 제안서회의 결과민원 분석
2
기관 보고서 형식 문서 확보
현재 사용 중인 공문서 서식이나 우수 보고서 샘플을 1~2건 준비합니다.
이를 지침에 포함하거나 파일로 첨부하면 AI가 형식을 즉시 학습합니다.
행안부 공문서 규정기관 보고서 샘플

[보고서 제목]: [기준일]: [핵심 수치]: [특이사항]: [담당부서]: # 입력이 불완전하면 빠진 항목을 먼저 물어보세요.
